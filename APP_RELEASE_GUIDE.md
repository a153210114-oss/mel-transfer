# 华伴 App 上架执行清单

目标：先把现有华伴 Web/PWA 包成 App，进入 Android 内测和 iOS TestFlight；稳定后再正式上架。

## 1. 推荐路线

### 第一阶段：Android 内测

- 用 Capacitor 把 `https://www.huabanapp.com/ai.html` 包成 Android App。
- 先走 Google Play 内部测试或封闭测试。
- 优点：速度快，方便发给公测用户安装。

### 第二阶段：iOS TestFlight

- 用同一套 Capacitor 壳打 iOS 包。
- 先走 TestFlight，不要直接写“公测版”提交正式 App Store。
- 等功能稳定、隐私页和举报机制补齐后，再提交正式 App Store。

### 第三阶段：正式上架

- App Store：正式版文案不要写“公测”，改成“早期体验 / 正式发布”。
- Google Play：可先发布到墨尔本/澳洲，再逐步开国家地区。

## 4. 商店简介草稿

华伴是面向海外华人的智能体 AI。

你可以把生活、事业、学习、出行、本地连接和情绪陪伴中的琐事告诉华伴。华伴会帮你整理信息、判断重点、生成可确认的事项，并在你需要时协助推进。

华伴不是一个让你迷失在信息流里的应用。它尽量保持干净、安静、少打扰。你需要时，它在；你不需要时，它不制造焦虑。

适合：

- 刚到海外，需要接送机、包车、本地生活帮助的人
- 已在海外生活，需要找服务、找资源、整理计划的人
- 有技能、商品或服务，希望被真正需要的人找到的人
- 想把熟人社交和本地连接分开，减少打扰的人
- 需要一个能聊天、能整理、能提醒、能帮忙推进事情的 AI 伙伴

## 5. 关键卖点

- 生活跟我聊：出行、提醒、行程、服务需求，华伴帮你整理。
- 生意跟我聊：你的强项、商品和服务，可以被真正需要的人看见。
- 情感跟我聊：能当树洞，也能当参谋。
- 本地圈子：微信联系原来的熟人圈，华伴连接当地朋友。
- 干净安静：没有一排排红点，不让算法牵着走。

## 6. 上架前必须补齐

### 隐私与安全

- `privacy.html`：说明收集哪些数据、用途、保存方式、删除方式。
- `terms.html`：用户协议。
- `support.html`：支持联系、反馈入口。
- 举报/拉黑机制：同城好友、名片、服务者信息必须能举报或屏蔽。
- 用户生成内容规则：同城连接、服务信息、名片和聊天内容要有社区规则。

### 权限说明

- 麦克风：语音输入、语音唤起。
- 相机/相册：识别截图、二维码、票据、行程信息。
- 通讯录：用户主动选择联系人邀请，不自动上传通讯录。
- 定位：用于判断本地城市、出行和本地服务范围。
- 通知：用于提醒、消息提醒、订单/工卡提醒。

### 支付说明

- 当前支付/打赏先作为“意向插件”，不自动扣款。
- 如果在 App 内售卖数字权益、订阅或打赏，需要分别按 Apple / Google 的支付规则处理。
- 线下真实服务或实物交易可走外部确认，但必须写清楚平台角色和风险边界。

## 7. 技术路线

### 推荐使用 Capacitor

1. 保留现有 Web 版本。
2. 新增 Capacitor 配置。
3. Android 用 Android Studio 打包测试。
4. iOS 用 Xcode 打包并上传 TestFlight。

拟定配置：

```json
{
  "appId": "com.huabanapp.ai",
  "appName": "华伴",
  "webDir": ".",
  "server": {
    "url": "https://www.huabanapp.com",
    "cleartext": false
  }
}
```

说明：第一版可以用线上地址作为 App 内容来源，更新快；正式稳定后再考虑把核心页面打包进 App。

## 8. 审核风险

- 纯 WebView 容易被认为功能不足，所以 App 内要体现手机能力：语音、扫码、图片识别、通知、添加好友、本地名片、提醒。
- 社交/同城功能必须有举报、屏蔽和安全提示。
- 专业服务必须注明不替代律师、会计、医生等专业判断，服务者资质需核验。
- “公测版 / beta”不要直接作为正式 App Store 文案；iOS 用 TestFlight。
- 涉及 AI 生成建议时，要说明用户需要自行确认重要事项。
