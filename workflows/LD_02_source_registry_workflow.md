# LD_02 中文公开信息入口库工作流

目标：把网站平台、搜索引擎、商家目录、政府公开网站和行业目录统一登记为入口库，让 Agent 按用户需求持续生成搜索任务，而不是临时全网乱搜。

## 入口类型

- `search_engine`：Google、Bing、DuckDuckGo 等，用于发现公开网页。
- `chinese_website`：当地中文门户、分类信息、招聘、黄页。
- `forum`：中文论坛、留学生论坛、城市社区。
- `social_platform`：Facebook 群、小红书、公众号等；默认人工授权或公开信息。
- `business_directory`：Google Maps、Yelp、Yellow Pages、本地华人商家目录。
- `government_registry`：政府公开执照、商业登记、专业资质查询。
- `industry_directory`：律师协会、会计协会、行业协会公开目录。
- `manual_channel`：只能由人工检查的渠道源。

## 每个入口必须记录

- 名称、网址、国家、城市、语言。
- 入口类型：需求、供给、资质核验、渠道源或两者都有。
- 访问方式：公开、需登录、人工授权、不可用。
- 抓取边界：公开信息、搜索结果、官方核验、人工核查或不抓取。
- 搜索模板：如 `{city} 华人 {keyword} {freshness}`。
- 合规备注：是否打码、是否需要登录、是否禁止自动抓取。
- 最近检查时间、优先级、状态。

## 用户需求进入搜索流水线

用户提出任意需求后，Agent 生成：

```text
城市 / 国家
需求类别
方向：需求或供给
关键词
时间窗口：需求90天内，供给2年内
是否需要资质核验
需要搜索的入口类型
```

然后从 `source_registry` 选择入口：

1. 先选当前城市/国家的中文网站和商家目录。
2. 再选全球同类城市入口做市场信号扫描。
3. 如果是律师、会计、电工、水管、房产中介等受监管服务，必须加入政府/行业公开核验入口。
4. 需要登录、加群、验证码、打码联系方式的入口，只生成后台人工任务。

## 分流

- 可直接使用的公开线索：进入 `beta_leads` / `recruitment_leads`，等待 review。
- 需要继续查的渠道：进入 `ld_01_daily_task`。
- 政府/行业公开网站：进入资质核验路径，不当作普通供给广告。
- 不合规或私密来源：不入库，只记录为不可用或人工备注。

## 禁止

- 不自动注册第三方网站账号。
- 不绕过登录、验证码、打码电话。
- 不自动加微信群、Facebook 群。
- 不抓私密群聊或私信。
- 不把 AI 猜测当真实联系方式。
- 不把广告页、黄页、自称资质当政府/行业核验结果。

## 后台

后台 `中文入口库` 读取 `source_registry`。如果表还没创建，先显示内置模板；执行 `source_registry_schema_seed.sql` 后改为读取数据库。
