# 华伴供给侧、匹配、合同化与 App 上架工作流

版本：2026-07-09

## 1. 供给侧录入

供给侧分两类：

- 主动搜索录入：华伴团队或 Agent 从公开网站、政府入口、行业目录、中文平台、商家官网、社群线索中发现资源，进入后台「录入供给」。
- 被动录入：用户、商家、司机、外卖员、专业人士主动告诉华伴“我能提供什么”，由 AI 引导补齐信息后进入审核。

供给侧字段尽量完整：

- 行业分类
- 国家、城市、服务范围
- 名称、联系人、联系方式
- 价格、收费方式、可接时间
- 服务/商品说明
- 资质、执照、公开核验链接
- 官网、店铺、菜单、作品集、社交主页
- ERP、电商网站或 API 接口
- 来源、推荐人、录入方式

## 2. 推荐关系闭环

推荐关系不是只记录一次邀请，而是形成闭环：

1. 用户提出需求。
2. 华伴匹配供给。
3. 供给方看到“通过华伴 AI 找到你”的真实需求。
4. 供给方愿意完善资料并成为供给侧。
5. 后续类似需求更快匹配。
6. 邀请人、供给方、成交记录和履约记录进入贡献关系。

记录字段建议：

- referrer_code
- source_user_id
- invited_user_id
- supply_lead_id
- demand_work_order_id
- conversion_stage
- contribution_points

## 3. 定位与精准匹配

匹配必须先识别：

- 用户当前城市/区域
- 服务类别
- 需求紧急程度
- 用户偏好权重

常见权重：

- 距离：附近、就近、离我近
- 价格：便宜、预算、性价比
- 速度：现在、尽快、着急
- 资质：靠谱、持牌、保险、正规
- 口碑：评分、评价、别人用过

当用户表现出选择困难，例如“你推荐”“哪个好”“随便”“帮我选”，华伴应主动推荐一个最合适对象，并简短说明理由。

类别不匹配的资源要静默跳过。找电工不能推荐司机、餐馆；找接送机不能推荐维修。

## 4. 交易合同化

所有涉及商品或服务成交的任务，都进入订单/工卡确认机制。

工卡是合同的一部分，包含：

- 时间
- 地点
- 任务
- 金额/预算
- 支付方式
- 备注
- 用户确认
- 服务方确认
- 条款版本
- 确认快照

双方必须分别勾选同意。双方都确认后，交易才生效。

履约证明可包括：

- 履约照片
- Google 地图位置
- 导航记录
- 聊天记录
- 到达/开始/完成时间
- 双方确认记录

高风险服务，例如法律、会计、医疗、电工等，必须保留资质核验说明和人工审核记录。

## 5. App 上架与系统通知

App 上架目标不是换壳，而是让华伴接入手机能力：

- 系统通知
- 来消息提醒
- 工卡状态提醒
- 提醒事项
- 扫码加好友
- 图片识别
- 相机上传
- 通讯录授权
- 本地名片

上线顺序：

1. 先保持 PWA 可添加到主屏幕。
2. 准备 iOS/Android WebView 或轻量原生壳。
3. 接入推送通知。
4. 接入扫码、相机、相册、通讯录权限。
5. 完成隐私政策、权限说明、App Store / Google Play 素材。
6. 提交审核。

当前阶段，系统通知先作为 App 上架任务跟踪，不在 Web 端假装已经接入原生通知。
